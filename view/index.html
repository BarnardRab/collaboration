Known Problems

  • The changing sizes of the helpbox text shifts the Menu Column up and down when elements are hovered over. This can be fixed with the css style="float: left;", but this introduces margin discrepencies.
  • The displays of dates take too much table space. Working with dates in JavaScript is a pain.
  • Filter is currently broken
  • The angled table headers are useful for conserving screen space without the confusion of abbreviations, but the edges partially cover the top row, making it difficult to click the first record.
  • When adding a new record to any of the listing pages, the cell widths will automatically adjust based on the contents.
  • The generateListingFunction(listingContents, listingTable) is intended to populate the listing pages of various record types such as Users, Projects and Work Orders with a table, navigation pane on the left and a create / filter section in that pane. The following problems exist regarding that:
    • The function fails on the statement menu.appendChild(navigation);
    • The navigation argument is generated by the statement: navigation = generateNavigation(listingContents.navigation)
    • For the example of the work orders listing, the argument, listingContents, is passed to the function generateListingPage(listingContents, listingTable) in the ../../view/listing/work_orders.html page
    • Navigation is defined by the statement: let navigation = {header: "Work Order Directory", links : [{item: "home", location: "../index.html", description: "Back to the landing page of the work order system"}]}
    • Navigation is added to the listingContents object in the statement: let listingContents = {idPrefix: "WorkOrder", focusedID: "WorkOrderRequested ByFooter", navigation: navigation};
